**Storage**: `internal_links` JSONB column on `blog_posts` table, shape `{path, anchorText}[]`.

**Analyzer reads**: `blogPostToMetadata` (line 480) reads `post.internal_links` directly from the JSONB field: `const internalLinks = Array.isArray(post.internal_links) ? post.internal_links : [];`. The quality analyzer (line 155) uses `metadata.internalLinks?.length || 0`.

**Fixer writes** (lines 760-767): After appending the HTML link block to content, the fixer merges new links into the JSONB field: `updatePayload.internal_links = [...existingLinks, ...newStructuredLinks]`.

**Verdict**: The persistence path is complete and correct. Both the HTML content and the JSONB field are updated in the same DB write (`updatePayload`). The analyzer reads from the JSONB field. **No code change needed.** The earlier plan language about it being "partly problematic" was wrong and is removed.

## FAQ Count Fallback: Exact Placement and Safety

**Placement**: Inside the existing FAQ append handler (lines 700-722), immediately after line 717 (after the `faqSchemaEligible` block closes), before the `fixesApplied.push` at line 719.

**Variable scope**: `sanitized` is declared at line 706 within the same handler block. It contains ONLY the FAQ HTML that was just appended — never non-FAQ content.

**Exact condition**: `if (!updatePayload.faq_count && contentChanged)` — runs only when (a) the primary faqSchema path did NOT set `faq_count`, AND (b) FAQ content was actually appended to the article.

**Counting method**: `(sanitized.match(/<h[3-4][^>]*>[^<]*\?/gi) || []).length` — counts only H3/H4 headings ending in `?` within the `sanitized` FAQ block. This cannot count non-FAQ content because `sanitized` is scoped to the FAQ HTML block only.

**Why safe**: The regex runs against `sanitized` (the FAQ block), not against `modifiedContent` (the full article). It cannot accidentally count conclusion headings, intro text, or other content.

```typescript
function detectIntro(content: string): boolean {
  const parser = new DOMParser();
  const doc = parser.parseFromString(content, 'text/html');
  const children = Array.from(doc.body.children);
  for (const child of children) {
    const tag = child.tagName.toLowerCase();
    if (/^h[1-6]$/.test(tag)) return false;
    if (tag === 'p' && (child.textContent?.trim().length || 0) > 20) return true;
    if (tag === 'div') {
      const inner = child.querySelector('p');
      if (inner && (inner.textContent?.trim().length || 0) > 20) return true;
      if (child.querySelector('h1,h2,h3,h4,h5,h6')) return false;
    }
  }
  return false;
}
```

No scoring thresholds, weights, or gates are changed. Same 20-char minimum. Same 5/2 readability scoring.
